# 版本閱讀指南

本講義的目的很單純：幫助在閱讀這個專案的「版本快照檔」時，不要被目前最新版的 import 鏈混淆。

## 1. 為什麼需要這份指南

這個專案不是每一版都放在獨立資料夾，而是把不同版本的重要檔案保留下來，例如：

- `backend_01.ts`
- `backend_02.ts`
- `backend_03.ts`
- `store/json/JsonFileStore_01.ts`
- `store/json/JsonFileStore_02.ts`

這種做法對教學很方便，因為可以直接對照不同版本的演進；但副作用是：

- 目前打開的是某個舊版檔案
- 它的 import 可能又連到目前正式版本的未加後綴檔案
- 讀到一半就不知道自己到底在看哪一版

所以，閱讀這份專案時，最好先知道：

> 每一個版本應該對應哪一組 `backend / contracts / Store / JsonFileStore`

## 2. 閱讀規則

### 基本原則

- 看某一版時，優先追該版對應的版本檔
- 不要直接把未加後綴的檔案當成所有版本的基準
- 未加後綴的 `backend.ts`、`contracts.ts`、`Store.ts`、`store/index.ts`，應視為目前版本或後續收斂版

### 實用判斷方式

若正在看某個版本，但看到下面情況，就要先停一下：

- 檔名是 `_02`，但卻 import `contracts.ts`
- 檔名是 `_03`，但卻 import `Store.ts`
- 講義說正在講 `V3`，但 source 已經連到 `V5` 的型別

這通常不是理解錯，而是 import 鏈把閱讀路徑帶到別的版本了。

## 3. 版本對照表

| 版本                | backend         | contracts                | store interface     | store index         | json store                                           | 說明                                                           |
| ------------------- | --------------- | ------------------------ | ------------------- | ------------------- | ---------------------------------------------------- | -------------------------------------------------------------- |
| V1 初版原型         | `backend_01.ts` | `shared/contracts_01.ts` | 無                  | 無                  | 無                                                   | 資料直接寫在 backend 內                                        |
| V2 資料持久化版     | `backend_02.ts` | `shared/contracts_01.ts` | `store/Store_01.ts` | `store/index_02.ts` | `store/json/JsonFileStore_01.ts`                     | 從記憶體版走向 JSON store                                      |
| V3 菜單內容升級版   | `backend_03.ts` | `shared/contracts_02.ts` | `store/Store_02.ts` | `store/index_03.ts` | `store/json/JsonFileStore_02.ts`                     | 新增 `description`、`image_url` 與 `normalizeMenuItem()`       |
| V4 體驗與部署修正版 | `backend_04.ts` | `shared/contracts_02.ts` | `store/Store_02.ts` | `store/index_04.ts` | `store/json/JsonFileStore_03.ts` → `04.ts` → `05.ts` | V4 是一組連續修正版，`index_04.ts` 預設指向最後收斂的 `_05.ts` |
| V5 目前版本         | `backend.ts`    | `shared/contracts.ts`    | `store/Store.ts`    | `store/index.ts`    | `store/json/JsonFileStore.ts`                        | 加入使用者、登入、訂單歸屬與歷史                               |

## 4. 建議追蹤鏈

若喜歡從入口一路追下去，可以直接照下面走。

-> store/index_02.ts
-> store/Store_01.ts
-> store/json/JsonFileStore_01.ts

## 5. 各版本的閱讀重點

### V1

重點是：

- 先把點餐流程跑通
- 資料直接放在 backend
- 還沒有 store abstraction

### V2

重點是：

- 把資料層抽出去
- 引入 `Store` 介面
- 引入 `JsonFileStore`

### V3

重點是：

- `MenuItem` 升級
- 新增 `description`
- 新增 `image_url`
- 用 `normalizeMenuItem()` 相容舊資料

### V4

重點是：

- 圖片資源與部署問題修正
- SPA fallback 行為修正
- 同一版中有多個 `JsonFileStore` 修正版

### V5

重點是：

- 引入 `User`
- 訂單開始綁定 `userId`
- 有登入、歷史訂單、送出訂單

## 6. 最後的建議

若閱讀目的為學版本演進，不要一開始就直接讀 `backend.ts`。

更好的方式是：

1. 先看講義目前在講哪一版
2. 再回這份指南查對應檔案
3. 然後沿著該版的追蹤鏈往下讀

這樣比較不會在不同版本的資料模型、API 契約與 store 實作之間來回跳動。
