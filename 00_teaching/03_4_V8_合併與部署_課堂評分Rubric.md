# V8 合併與部署調整課堂評分 Rubric

本 Rubric 對應 V8 完成後的教學任務：分支驗證、PR 審查、合併主線、部署調整與 CI/CD。

---

## 1. 評分原則

### 為什麼

課堂評分若只看功能有沒有跑起來，會忽略工程流程能力。此 Rubric 用來評估是否具備可部署的團隊開發習慣。

### 要做什麼

以 100 分制評估，採五個面向。

---

## 2. 評分面向與配分

1. 分支品質閘門（20 分）
2. PR 品質與審查紀律（25 分）
3. 主線合併與版本治理（20 分）
4. Render 部署調整驗證（20 分）
5. CI/CD 最小自動化（15 分）

---

## 3. 等第描述

### A 級（90-100）

- 流程完整，證據清楚，可重現
- 能說明每一步背後原因，不只是照指令操作
- merge 與部署後，主線穩定可用

### B 級（80-89）

- 主要流程完成
- 少量證據不足或文件不夠完整
- 不影響整體可部署性

### C 級（70-79）

- 流程有完成但品質不穩
- 多處缺乏驗證證據或核可紀錄
- 需要助教提醒才能完成收斂

### D 級（60-69）

- 只完成部分步驟
- 無法證明主線可穩定部署

### 未達標（0-59）

- 缺乏基本流程觀念
- 無法完成主線合併或部署驗證

---

## 4. 細項評分標準

### 4.1 分支品質閘門（20 分）

- 20 分：build、測試與必要檢查完整，結果有證據
- 15 分：build 完成但檢查項不完整
- 10 分：僅口頭說明，缺乏可驗證紀錄
- 0 分：未進行或失敗未處理

### 4.2 PR 品質與審查紀律（25 分）

- 25 分：PR 模板欄位完整，Reviewer 與 PM/PO 核可齊全
- 20 分：PR 內容完整但核可紀錄缺一項
- 15 分：PR 描述過於簡略，風險與驗證證據不足
- 0 分：無 PR 或未依流程審查

### 4.3 主線合併與版本治理（20 分）

- 20 分：正確合併到 main，且有 tag 或等效版本標記
- 15 分：成功合併但版本治理資訊不足
- 10 分：合併流程混亂，仍可勉強追蹤
- 0 分：未回到 main 或流程不可追蹤

### 4.4 Render 部署調整驗證（20 分）

- 20 分：雲端健康檢查、核心 API 與資料庫調整均驗證成功
- 15 分：服務啟動成功但驗證項目不足
- 10 分：僅完成部署，缺乏可驗證回應證據
- 0 分：部署失敗或無驗證

### 4.5 CI/CD 最小自動化（15 分）

- 15 分：PR 與 main push 均可自動觸發 CI，結果可查
- 10 分：僅完成單一路徑觸發
- 5 分：有 workflow 檔但不可用
- 0 分：未建置 CI

---

## 5. 助教快速評分流程

1. 先看 PR 內容是否使用模板且欄位完整
2. 再看 CI 結果是否可追溯
3. 再看 main 是否有合併紀錄與版本標記
4. 最後抽查 Render 健康檢查與至少一支 API 回應

建議：每組評分時，同步記錄一項做得最好與一項需改進，作為下次迭代回饋。
